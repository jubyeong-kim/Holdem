# ♠ Menti Hold'em ♥

Mentimeter 방식의 실시간 멀티플레이 텍사스 홀덤 웹앱.
호스트가 PC로 방을 열면, 참가자들이 폰으로 QR·코드를 통해 접속해서 함께 플레이한다.

> 📌 **본 프로젝트는 「2. [프로젝트] 백엔드 — 화면 뒤에서 일어나는 일들」 과제 제출을 위해 만들어졌습니다.**

## 🔗 라이브 데모

**https://holdem-bu59.onrender.com**

- 호스트: `/login.html` (또는 첫 화면의 "호스트로 방 열기")
- 게스트: `/player.html` — 참가 코드 입력 또는 호스트 화면 QR 스캔

> ⚠️ 무료 호스팅(Render)이라 15분간 접속이 없으면 서버가 잠듭니다. 첫 접속 시 깨어나는 데 최대 ~50초 걸릴 수 있으니, 화면이 안 뜨면 잠시 후 새로고침해 주세요.

## 기능

- **실시간 멀티플레이** — WebSocket(socket.io), 방 코드로 접속, 재접속 처리
- **QR 참가** — 호스트 화면의 QR을 스캔하면 코드 자동 입력
- **2가지 조작 모드**
  - 호스트가 폰으로 전원 베팅 조작 (컨트롤러 화면)
  - 게스트가 각자 폰에서 직접 베팅
- **카드 피크(squeeze)** — 게스트가 홀카드를 밑에서 위로 밀어올려 확인, 떼면 다시 뒤집힘
- **판돈(블라인드) 설정** — 설정 화면 및 핸드 사이에 변경 (10/20 ~ 500/1000)
- **호스트 로그인** — 아이디/비밀번호 (scrypt 해시, 서버측 방 생성 인증)
- **초보자용 룰 팝업** — 게스트 화면에서 게임 방법 확인
- **족보 판정** — `pokersolver`

## 실행

```bash
npm install
npm start        # http://localhost:3000
node test.js     # 엔진/인증 self-check
```

- **PC(호스트)**: `http://localhost:3000` → 로그인 → 조작모드 선택 → 방 열기
- **폰(게스트)**: 같은 WiFi에서 `http://<PC-LAN-IP>:3000` → 코드 입력 또는 QR 스캔
- **폰(호스트 조작)**: `http://<PC-LAN-IP>:3000/control.html` (호스트 조작 모드일 때)

## 구조

| 파일 | 역할 |
|---|---|
| `poker.js` | 홀덤 룰 엔진 (덱·베팅라운드·족보·팟분배) |
| `server.js` | socket.io 서버, 방(room) 관리, QR 생성 |
| `auth.js` | 호스트 계정 (scrypt 해시, 세션) |
| `public/host.html` | 호스트 PC 화면 (설정 → 로비 → 진행) |
| `public/player.html` | 게스트 폰 화면 (홀카드 피크, 액션, 룰) |
| `public/control.html` | 호스트 폰 컨트롤러 |
